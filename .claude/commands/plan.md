## Planning Protocol
1. **Research** — Read relevant files, understand existing patterns
2. **Analyze** — Identify constraints, dependencies, risks
3. **Design** — Outline the implementation approach
4. **Break down** — List discrete steps in order
5. **Estimate complexity** — Flag high-risk or ambiguous steps

Output a structured plan before taking any action. Wait for approval before implementation.
